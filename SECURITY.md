# Security Policy

## Supported Versions

Currently, only the latest release of Sprout is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.1.x | :white_check_mark: |
| < 0.1.x  | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Sprout, please do **NOT** open a public issue. Instead, please send an email privately to the maintainer at hello@jessn.dev. 

All security vulnerabilities will be promptly addressed. You should expect a response within 48 hours.

## Scope
Please note that this security policy applies to the **Sprout CLI engine itself**, not to the Spring Boot dependencies or architectural templates it generates. 

Security vulnerabilities in the generated Java/Kotlin code should generally be routed upstream to the Spring Framework or the respective open-source project (like Testcontainers or springdoc-openapi).
